# Progressive Overload & Periodization

When designing a workout program, use the principles below. State the
mechanism explicitly in the program you produce — the user should be able
to see *why* each block is structured the way it is.

## 1. Progressive overload — the mechanism

Progress must be measurable, gradual, and tied to one variable at a time.
Each session, attempt **one** of:

- More weight (preferred for strength goals)
- More reps at the same weight
- One additional set
- Shorter rest periods (hypertrophy / conditioning goals)
- Better technique (always, before adding load)

Targets for week-over-week increase on compound lifts:

- Beginner (≤ 6 months trained): +2.5 kg upper body, +5 kg lower body
- Intermediate: +1–2 % per week
- Avoid jumps > 10 % week-over-week — injury risk outweighs gains

## 2. Periodization — structure the program in cycles

Do not write a static, week-after-week-identical plan. Use blocks:

| Phase | Volume | Intensity | Goal |
|---|---|---|---|
| Accumulation (weeks 1–3) | high | low–moderate | Build work capacity, technique |
| Intensification (weeks 4–6) | moderate | high | Build strength / skill |
| Realization (week 7) | low | very high | Test maxes / peak |
| Deload (week 8) | very low | very low | Active recovery |

State the phase for every week of the program you output.

## 3. Volume landmarks (sets per muscle group, per week)

- **MEV** (minimum effective volume): ~8–10 hard sets
- **MAV** (max adaptive volume): ~12–18 hard sets
- **MRV** (max recoverable volume): ~20+ hard sets — short-term only

Default to **MEV–MAV**. Push toward MRV only during a planned
intensification block, then deload.

## 4. Exercise selection

- 60–70 % of weekly volume on compound lifts (squat, hinge, press, pull, carry)
- 30–40 % on accessory / isolation work
- Always prioritize joint tolerance and injury history over "optimal" choice — a
  squat the user can do pain-free beats a "better" squat they can't

## 5. Time-budget scaling

Scale the program to the user's stated weekly time budget:

| Hours / week | Structure |
|---|---|
| ≤ 3 | 2 full-body sessions, compound-only |
| 4–5 | 3 full-body sessions, compound + light accessories |
| 6–8 | 4-day upper / lower split |
| 9+ | Push / pull / legs or body-part split |

## 6. Deload triggers

Recommend an early deload (drop to "Deload" row of the table in §2) when
the user reports **any two** of:

- Sleep quality drop for 5+ consecutive days
- Reps-in-reserve creeping down on the same load
- Persistent joint discomfort
- Loss of motivation / dread before sessions

## How to apply when generating a program

- Name the periodization **phase** for each week.
- Show the **progressive-overload mechanism** explicitly (e.g. "+2.5 kg
  week-over-week on main lifts; reps held at 5").
- Stay at **MEV–MAV** unless the user is past beginner stage.
- Match **session count** to the time budget in §5.
- For users with no specified limitations, still include **deload trigger
  guidance** in the program notes so they know when to back off.
